# Nurturing Your Agent — como adoptar

## O que é isto

Um guião curto sobre **como afinar um agente de IA pela interacção quotidiana**. Em vez de tentar dar-lhe uma "instrução-mãe" perfeita, o agente aprende ao longo do tempo a partir do feedback do operador.

Originalmente desenvolvido num contexto pessoal de gestão de portfólio multi-vertente (clínica + académica + empreendedorismo). O método é genérico — aplica-se a qualquer relação operador-agente longa em que o agente precisa de aprender convenções, valores, e padrões do operador.

## Conteúdo

- `nurturing-your-agent_pt-pt.md` — versão portuguesa (PT-PT)
- `nurturing-your-agent_en.md` — versão inglesa (EN)

Os dois ficheiros são **conteúdo paralelo** — escolhe o que preferires. Sem dependências externas.

## O que vais encontrar

1. **Os 3 canais de afinação** (feedback, memórias, regras)
2. **Tipos de memória** (perfil, preferências, projecto, padrões)
3. **Como capturar** correcções e validações sem perder coerência
4. **Casos práticos** (10+ exemplos reais com gatilho, regra, memória)
5. **Antipadrões** comuns (e como os evitar)

## Como adoptar (5 passos)

1. **Escolhe a tua plataforma** — Claude Code, Claude.ai com Custom Instructions, ChatGPT com Memory, Cursor, ou outro agente IA com memória persistente
2. **Lê o guião** uma vez de seguida (~25 min)
3. **Cria a tua estrutura de memória** — pasta dedicada com ficheiros markdown indexados (ver §3 do guião)
4. **Captura feedback à medida que ele aparece** — quando corriges o agente, regista a correcção como memória; quando ele acerta um padrão não-óbvio, regista também
5. **Revisão periódica** — uma vez por semana, vê quais memórias estão activas, quais ficaram obsoletas, quais devem ser promovidas

## O que NÃO está incluído

- Código específico de plataforma (cada agente tem o seu mecanismo de memória)
- Templates de prompt master
- Configuração de hooks/automações

→ Estas decisões ficam ao critério de cada operador. O guião dá o **método**; tu adaptas à tua **stack**.

## Próximos passos típicos

Se este guião te poupar tempo de exploração, considera:

- **Documentar os teus próprios padrões** com a mesma estrutura (gatilho → regra → memória) — vais ter um corpus reutilizável
- **Partilhar de volta** — se descobrires antipadrões que aqui não estão, contribuir para uma v1 melhor é bem-vindo
